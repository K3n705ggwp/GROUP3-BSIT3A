Queries

Key SQL tables and their relationships in the Parcel Lockbox database

Below are the essential tables and relationships used in our parcel lockbox system. Each table serves a role in managing rentals, customers, and lockboxes, forming a relational database designed for secure and efficient tracking.

1. Landlord Table

This table stores details about the lockbox owner or administrator who registers customers and lockboxes.

CREATE TABLE `landlord` (
  `landlordID` INT AUTO_INCREMENT,
  `reg_Customer` VARCHAR(50),
  `reg_Rental` VARCHAR(50),
  PRIMARY KEY (`landlordID`)
);

Primary Key: landlordID

Used by: References in lockbox and customer tables.

2. Lockbox Table

Represents each physical lockbox and stores its location and status (e.g., available, rented).

CREATE TABLE `lockbox` (
  `lockboxID` INT AUTO_INCREMENT,
  `landlordID` INT,
  `location` VARCHAR(100),
  `status` VARCHAR(20),
  PRIMARY KEY (`lockboxID`),
  FOREIGN KEY (`landlordID`) REFERENCES `landlord`(`landlordID`)
);

Foreign Key: landlordID (from Landlord)

Relationship: Many lockboxes can belong to one landlord (1:N).

3. Customer Table

Holds customer profiles who rent lockboxes, including their name, contact, and address.

CREATE TABLE `customer` (
  `customerID` INT AUTO_INCREMENT,
  `landlordID` INT,
  `lname` VARCHAR(50),
  `fname` VARCHAR(50),
  `mi` CHAR(1),
  `email` VARCHAR(100),
  `contactNo` INT(20),
  `address` VARCHAR(255),
  PRIMARY KEY (`customerID`),
  FOREIGN KEY (`landlordID`) REFERENCES `landlord`(`landlordID`)
);

Foreign Key: landlordID (from Landlord)

Relationship: A landlord can manage multiple customers (1:N).

4. Rental Table

This table links customers to the lockboxes they rent. It also stores rental dates and rental status.

CREATE TABLE `rental` (
  `rentalID` INT AUTO_INCREMENT,
  `customerID` INT,
  `lockboxID` INT,
  `rentalStartDate` DATE,
  `rentalEndDate` DATE,
  `rentalStatus` VARCHAR(20),
  PRIMARY KEY (`rentalID`),
  FOREIGN KEY (`customerID`) REFERENCES `customer`(`customerID`),
  FOREIGN KEY (`lockboxID`) REFERENCES `lockbox`(`lockboxID`)
);

Foreign Keys: customerID (from Customer), lockboxID (from Lockbox)

Relationship: This is a bridge table. It creates a many-to-many (M:N) relationship between customers and lockboxes through rentals.

Entity Relationships Summary

  • Landlord → Lockbox: One landlord can own many lockboxes (1:N)
  • Landlord → Customer: One landlord can manage multiple customers (1:N)
  • Customer → Rental: One customer can have multiple rentals (1:N)
  • Lockbox → Rental: One lockbox can be rented many times (1:N)
  • Rental: Connects customer and lockbox (M:N resolved using Rental)
