Database Architecture

Our Parcel Lockbox system uses a relational database designed for security, traceability, and scalability. The architecture supports transaction integrity, multi-user operations, and compartment-based data access.

Centralized Database Architecture (With Admin Functionality)

This architecture diagram illustrates a centralized database system where all entities—clients, database, and admin—interact with a single, central server. It ensures unified data access, consistent operations, and simplified system management.

Components:

  1. Clients (Users)
    Represent end-users such as customers who access system features like rental services and parcel delivery information. Clients send and receive data through API calls to the central database.
  2. Central Database Server
    A single, unified storage that houses all critical data including customer records, lockbox status, rental details, delivery logs, and payment transactions.
    • -Ensures real-time access, data consistency, and easier maintenance.
    • -Designed with role-based access control (RBAC) to separate access privileges by user roles.
  3. Admin (Landlord)
    Acts as the primary controller of the system:
    • -Registers new customers
    • -Manages lockbox availability and assignments
    • -Oversees rentals and updates statuses
    • -Monitors deliveries and payments

The landlord interacts directly with the database, ensuring all operational data is current and properly managed. This role has elevated privileges compared to regular users.

Key Benefits:

  • Single Source of Truth – All data operations are performed in one place, minimizing redundancy.
  • Real-Time Operations – Immediate updates to lockbox and parcel status ensure users and admins work with the most recent data.
  • Secure and Scalable – Supports encryption, access controls, and is ideal for small to mid-sized deployment zones.

Key Components

User Management

Handles Admin and Customer information, including login credentials and contact data.

Parcel Tracking

Tracks parcels with details like size, origin, destination, and delivery attempts.

Rider Info

Stores rider name, contact number, and courier company for accountability and coordination.

Payment System

Supports both Cash on Delivery and digital payment methods like GCash and PayMaya.

PIN Access

Securely manages auto-generated PIN codes for parcel retrieval by authorized users only.

Compartment Leasing

Tracks compartment rental records, including start/end time and assigned users.
